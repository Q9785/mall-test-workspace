# 测试范围与策略

## 一、测试目标

验证 mall 商城 B端后台管理及 C端前台交易核心链路的正确性，覆盖功能、接口、自动化、性能、安全、弱网等维度。

## 二、测试范围

### 2.1 功能测试

- **B端后台**：登录、商品管理、订单管理、售后管理
- **C端前台**：登录/注册、商品搜索/详情、购物车、下单/支付、秒杀

### 2.2 接口测试

- **B端**：登录、新增商品、上架、列表查询、订单发货、同意退款
- **C端**：登录、注册、加购、下单、支付、确认收货

### 2.3 自动化测试

- **框架**：Python + Pytest + Requests
- **覆盖**：登录成功、登录失败、登录后加购
- **报告**：使用 Allure 生成自动化测试报告

### 2.4 性能测试

- **工具**：JMeter
- **场景**：秒杀接口并发（100用户）、加购接口并发（100用户）、下单接口并发（100用户），混合接口并发（100用户）
- **结果**：0% 异常率

### 2.5 安全测试

- SQL注入、越权、价格篡改、明文密码传输

### 2.6 弱网测试

- 模拟 3G/4G 网络、30% 丢包率下加购防重复提交

## 三、优先级策略

- **P0**：核心业务流程（登录→加购→下单→支付），100% 覆盖
- **P1**：重要功能（搜索、筛选），尽量覆盖
- **P2**：边缘功能（统计、导出），设计用例但视时间执行
